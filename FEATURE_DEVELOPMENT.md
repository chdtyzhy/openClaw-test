# 功能开发流程指南

## 原则
- **一次一个功能**：避免多任务导致混乱
- **本地验证**：确保功能在本地完全工作
- **及时推送**：完成即推送，避免环境差异
- **小步提交**：频繁提交，便于回滚和追踪

## 标准工作流

### 1. 准备阶段
```bash
# 确保在干净的 dev 分支
git checkout dev
git pull origin dev
git status  # 确保没有未提交更改

# 创建功能分支
git checkout -b feature/简短描述
# 示例：feature/history-view, feature/percentage-calc
```

### 2. 开发阶段
```markdown
## 功能卡片
- **功能名称**: [明确描述]
- **目标**: [要解决的问题]
- **验收标准**:
  1. [可测试的条件1]
  2. [可测试的条件2]
  3. [可测试的条件3]
- **预计工时**: [小时数]
```

### 3. 实现阶段
```swift
// 代码结构建议
// 1. 先设计数据模型（Model）
// 2. 实现业务逻辑（ViewModel）
// 3. 创建界面（View）
// 4. 添加测试（可选但推荐）
```

### 4. 验证阶段
```bash
# 本地测试清单
- [ ] 编译通过，无警告
- [ ] 功能按预期工作
- [ ] 界面显示正确
- [ ] 边缘情况处理
- [ ] 性能可接受

# 在 Xcode 中测试
- 运行应用 (⌘R)
- 测试不同设备尺寸
- 测试横竖屏切换（如有需要）
```

### 5. 提交阶段
```bash
# 提交代码
git add .
git commit -m "feat: 简短描述功能

- 具体更改1
- 具体更改2
- 具体更改3"

# 推送到远程
git push origin feature/分支名称

# 创建 Pull Request（可选）
# 在 GitHub 上创建 PR 从 feature/* 到 dev
```

### 6. 合并阶段
```bash
# 回到 dev 分支
git checkout dev
git pull origin dev

# 合并功能分支
git merge feature/分支名称

# 解决冲突（如果有）
# 再次测试合并后的代码

# 推送到远程 dev
git push origin dev

# 删除本地功能分支
git branch -d feature/分支名称

# 删除远程功能分支（可选）
git push origin --delete feature/分支名称
```

## 功能分支命名规范
```
feature/功能名称           # 新功能
fix/问题描述              # 问题修复
refactor/重构描述         # 代码重构
docs/文档更新             # 文档更改
style/代码风格            # 代码格式
test/测试相关             # 测试代码
```

## 提交信息规范
```
feat: 添加新功能
fix: 修复问题
docs: 更新文档
style: 代码格式
refactor: 代码重构
test: 添加测试
chore: 构建过程或辅助工具
```

## 功能开发检查清单

### 开始前
- [ ] 阅读相关文档
- [ ] 理解需求
- [ ] 设计解决方案
- [ ] 评估复杂度

### 开发中
- [ ] 小步提交
- [ ] 保持代码整洁
- [ ] 添加注释（复杂逻辑）
- [ ] 遵循现有代码风格

### 完成后
- [ ] 本地完整测试
- [ ] 代码审查（如有条件）
- [ ] 更新文档
- [ ] 清理临时文件

## 环境一致性提示
1. **始终从最新 dev 开始**
2. **使用相同的 Xcode 版本** (26.2)
3. **相同的 iOS 目标版本** (17.0+)
4. **避免环境特定配置**
5. **使用配置模板而非硬编码**
